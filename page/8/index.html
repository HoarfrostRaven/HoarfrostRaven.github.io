IntroDocker 可以简化应用程序的配置、发布和测试，它彻底释放了计算虚拟化的威力，极大提高了应用的维护效率，降低了云计算应用开发的成本。 使用 Docker 的目的是解决因为开发环境不同而导致的程序开发或运行时会遇到的各种问题。举个例子说明，我们开发了一个应用程序，它在我们的电脑上可以正常运行，此时我们想把它分享给我们的朋友，那朋友不仅需要下载我们的代码，还需要配置相同的环境（插件、库等等）。即便如此，程序也不一定能正常运行，因为朋友使用的操作系统也可能与我们的不同，即便是同样的操作系统，也可能因为版本不同而导致程序无法正常运行。此时我们就需要 Docker 来帮助我们模拟完全相同的开发环境。 这里需要提到 Docker 和虚拟机的区别。它们都可以帮助我们模拟相同的开发环境，但是虚拟机需要先模拟硬件，然后在其上运行一个完整的操作系统，之后再在这个系统上运行所需的应用进程。这导致虚拟机不但体积臃肿、内存占用高，而且会影响程序的性能。Docker 在概念上与虚拟机类似，但比虚拟机更加轻便快捷。Docker 不会去模拟底层的硬件，它只会为每一个应用提供完全隔离的运行环境，我们可以在每 ...
